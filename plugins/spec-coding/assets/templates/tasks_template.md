## 执行规则

1. **严格顺序执行：** 从上到下，一次只处理一个 `- [ ]` 复选框
2. **单任务约束：** 每个复选框完成后必须经过验证，才可标记为 `- [x]`
3. **禁止跳跃：** 不得跳过任何任务，除非人类明确指示 "跳过"（标记为 `- [~]`）
4. **退回机制：** 如发现需要修改 `product.md`、`architecture.md` 或 `tasks.md`，必须暂停实现、更新规范、重新验证并重新获得批准
