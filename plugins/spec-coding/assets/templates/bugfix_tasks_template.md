## 执行规则

1. **先证据，后修复：** 第一项任务优先建立自动化复现或最强可用证据
2. **最小变更：** 只修改解决当前问题所必需的代码路径
3. **必须防回归：** 至少保留一项验证“不变行为未被破坏”的任务
4. **严格顺序执行：** 从上到下，一次只处理一个 `- [ ]` 复选框
5. **退回机制：** 如果根因判断变化，必须暂停修复、更新 `bugfix.md` / `design.md` / `tasks.md`、重新验证并重新获得批准
