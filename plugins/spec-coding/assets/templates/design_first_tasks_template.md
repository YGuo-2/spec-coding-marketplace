## 执行规则

1. **设计优先：** 任务必须首先满足 `design.md` 中已批准的约束与边界
2. **需求从设计派生：** 若 `requirements.md` 与 `design.md` 冲突，先暂停实现、回退修改文档、重新验证并重新获得批准
3. **严格顺序执行：** 从上到下，一次只处理一个 `- [ ]` 复选框
4. **单任务约束：** 每个复选框完成后必须经过验证，才可标记为 `- [x]`
5. **禁止越界：** 不得实现未在 `design.md` 明确支撑的能力
